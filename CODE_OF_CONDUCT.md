# Contributor Code of Conduct

## Our Standard

JOG is a technical project, but normal professional standards still apply.

People contributing here are expected to be direct, respectful, and factual. Disagreement on design, scope, or implementation is normal. Personal attacks, harassment, bad-faith disruption, and repeated low-signal behavior are not acceptable.

Examples of expected behavior:

- keep critique focused on code, behavior, tests, docs, and design tradeoffs
- state disagreements concretely
- respect maintainers' time by reading the current docs before proposing broad changes
- handle partial or pre-release areas honestly instead of overstating what the project can do

Examples of unacceptable behavior:

- personal attacks or insulting language
- harassment, discrimination, or intimidation
- spam, trolling, or repeated bad-faith derailment
- knowingly misrepresenting the project, another contributor's work, or the current implementation state

## Scope

This code of conduct applies to:

- GitHub issues
- pull requests
- discussions
- review comments
- project-linked chat or coordination spaces if they are created later

## Enforcement

Maintainers may edit, remove, or reject contributions and comments that do not meet this standard.

Repeated or serious violations may lead to temporary or permanent removal from project participation spaces.

## Reporting

For conduct concerns, contact the maintainers privately instead of escalating the issue in a public thread.

Use the contact guidance in [SECURITY.md](SECURITY.md) if the concern also involves a security-sensitive report.
