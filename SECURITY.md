# Security Policy

## Supported Status

JOG is currently pre-release software published as a technical preview.

That means the security posture is still evolving. The repo is usable for evaluation and contribution, but it should not be described as hardened for high-risk production use.

## Reporting A Vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Instead, report it privately to the maintainers with:

- a short description of the issue
- affected files or runtime area
- reproduction steps or proof of concept
- impact assessment if you have one
- any proposed mitigation if already known

If a private security reporting channel is added later, this file should be updated to point to it directly.

## Response Expectations

Maintainers will try to:

- acknowledge receipt
- confirm whether the issue is reproducible
- decide whether a fix, mitigation, or public advisory is needed

Because JOG is still pre-release, response times and remediation windows are not yet governed by a formal SLA.
